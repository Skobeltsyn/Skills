# 5 — Tasks (implementation)

Concrete implementation/build tasks derived from the specs (`../3-specs/`) and
designs (`../4-design/`). This is where the workflow turns validated design into
engineering work ready to be built.

An item enters this stage once its spec and design components exist upstream,
and leaves it when what it describes is built in `../6-results/`.

## Each task should have
- Link back to its spec (`../3-specs/…`) and/or design (`../4-design/…`)
- Clear scope and acceptance criteria (definition of done)
- Implementation notes / dependencies
