# Actors

Who/what interacts with the system — the **who**.

User roles, personas, and external systems. Define each actor's identity,
goals, and permissions.

Per `AGENTS.md`, each actor is a separate spec named `ACTOR-{number}-NAME`.
An actor's file lists all its connections (via entities), all its events, and
all its use-cases as links.
